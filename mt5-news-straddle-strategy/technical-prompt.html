Este es el prompt tecnico listo para pasarselo a un programador MQL5 o a una IA generadora de codigo. Copia el contenido completo del bloque inferior y utilizalo como especificacion de desarrollo.

Arquitectura Modular del EA
Modulo (Archivo) Responsabilidad Detalle Tecnico
NewsStraddleEA.mq5 Motor Principal Handlers de eventos: OnInit, OnTick, OnTimer, OnChartEvent, OnDeinit. Orquesta la maquina de estados: IDLE → ARMED → MONITORING → TRADE_ACTIVE → IDLE
Inputs.mqh Parametros de Entrada Todos los input configurables: InpNewsTime, InpEntryDistPips, InpSLPips, InpTPPips, InpEnableTP, InpTrailingStopPips, InpLotSize, InpMagicNumber, InpLeadTimeSeconds.
Globals.mqh Estado y Utilidades Enum ENUM_EA_STATE, variables globales (g_state, g_trade, g_posInfo, niveles buy/sell), funciones utilitarias: PipsToPrice(), InitPipFactor(), ParseEventTime(), StateToString().

Mejores Practicas Exigidas
  • CRITICO Usar exclusivamente la API CTrade y las clases estandar de MQL5 (Trade/Trade.mqh, Trade/PositionInfo.mqh).
  • CRITICO Validar SIEMPRE los resultados de g_trade.Buy()/Sell() con ResultRetcode() y manejar errores.
  • CRITICO Normalizar todos los precios con NormalizeDouble(price, _Digits) antes de operar.
  • IMPORTANTE Usar input variables para todos los parametros configurables.
  • IMPORTANTE Implementar un Magic Number unico para identificar las operaciones del EA.
  • BEST PRACTICE Arquitectura modular con .mqh separados (Inputs, Globals, UI, Orders).
  • BEST PRACTICE Separar la logica de UI del motor de trading. El panel solo lee y dispara estados, nunca ejecuta ordenes directamente.
  • BEST PRACTICE Cleanup completo en OnDeinit(): eliminar objetos graficos, kill timer, liberar recursos.
Prompt de Implementacion (Copiar Completo)
ROL Y CONTEXTO Eres un programador experto en MQL5 para MetaTrader 5. Tu especialidad es crear Expert Advisors (EAs) de grado profesional, robustos, eficientes y seguros para operar en condiciones reales de mercado, incluyendo eventos de alta volatilidad como noticias economicas. OBJETIVO Desarrollar un Expert Advisor (EA) modular en MQL5 llamado NewsStraddleEA que implemente la estrategia "Pre-News Market Order Straddle". El EA debe ser instalable directamente en MT5, compilar sin errores ni warnings, y funcionar en cuentas demo y reales. ESTRATEGIA (LOGICA DE NEGOCIO) 1. El usuario configura la hora exacta del evento/noticia en formato HH:MM:SS (hora del servidor MT5). 2. Exactamente InpLeadTimeSeconds (default 10) segundos antes del evento, el EA calcula dos niveles de entrada: - BuyLevel = Ask + PipsToPrice(InpEntryDistPips) — nivel de compra arriba del precio - SellLevel = Bid - PipsToPrice(InpEntryDistPips) — nivel de venta abajo del precio 3. El EA monitorea el precio tick a tick: - Si Ask >= BuyLevel: ejecuta una orden BUY de mercado con SL (y TP si habilitado) - Si Bid <= SellLevel: ejecuta una orden SELL de mercado con SL (y TP si habilitado) - Solo se ejecuta UNA de las dos 4. Una vez abierta la posicion, el trailing stop mueve el SL siguiendo al precio (solo cuando la posicion alcanza breakeven o mejor). 5. La posicion se cierra por SL, TP, trailing, o manualmente con CANCEL ALL. MAQUINA DE ESTADOS (STATE MACHINE) El EA opera con los siguientes estados estrictos: - STATE_IDLE: Sin actividad. EA en reposo. - STATE_ARMED: Evento programado. Countdown T-MM:SS visible. Esperando lead time. - STATE_MONITORING: Niveles calculados, lineas en chart. Vigilando precio tick a tick. - STATE_TRADE_ACTIVE: Market order ejecutada. Trailing stop activo. Esperando cierre. Transiciones: IDLE -> (OnInit) -> ARMED ARMED -> (T-LeadTimeSeconds alcanzado) -> MONITORING MONITORING -> (precio cruza nivel) -> TRADE_ACTIVE TRADE_ACTIVE -> (posicion cerrada) -> IDLE Cualquier estado -> (boton CANCEL) -> IDLE (cleanup total) PARAMETROS DE ENTRADA (input variables) input string InpNewsTime = "08:30:00"; // Hora del evento (HH:MM:SS, hora del servidor) input double InpEntryDistPips = 1000.0; // Distancia de entrada en pips input double InpSLPips = 10000.0; // Stop Loss en pips input double InpTPPips = 10000.0; // Take Profit en pips input bool InpEnableTP = true; // Activar Take Profit input double InpTrailingStopPips = 2000.0; // Trailing Stop en pips (0=off) input double InpLotSize = 0.10; // Tamano de lote input int InpMagicNumber = 202601; // Magic Number unico del EA input int InpLeadTimeSeconds = 10; // Segundos antes del evento para monitorear CONVERSION DE PIPS Funcion PipsToPrice(double pips) que convierte pips a precio: - Para pares Forex con 5 o 3 digitos: pipFactor = 10.0 * _Point - Para otros: pipFactor = _Point Usar _Digits para determinar automaticamente. NUNCA hardcodear el factor. LOGICA DE ENTRADA (modulo Orders) 1. SetStraddleLevels(): calcula BuyLevel, SellLevel, y sus respectivos SL/TP. 2. DrawTriggerLines(): dibuja lineas horizontales en el chart (verde=buy, roja=sell). 3. En OnTick() durante STATE_MONITORING: - Si Ask >= g_buyLevel: g_trade.Buy(InpLotSize, _Symbol, ask, g_buySL, g_buyTP) - Si Bid <= g_sellLevel: g_trade.Sell(InpLotSize, _Symbol, bid, g_sellSL, g_sellTP) - Verificar ResultRetcode(). Si OK -> STATE_TRADE_ACTIVE. Si error -> STATE_IDLE. - Eliminar lineas de trigger al ejecutar. TRAILING STOP (modulo Orders) ApplyTrailingStop() — llamado en cada tick cuando STATE_TRADE_ACTIVE: - Si InpTrailingStopPips <= 0: no hacer nada - Para BUY: newSL = bid - trailDist. Solo mover si newSL >= openPrice (breakeven) Y newSL > currentSL - Para SELL: newSL = ask + trailDist. Solo mover si newSL <= openPrice (breakeven) Y newSL < currentSL - Usar g_trade.PositionModify() para actualizar SL GESTION DE POSICION ACTIVA En cada OnTick(), si STATE_TRADE_ACTIVE: - Buscar posicion por Magic Number (loop PositionsTotal, filtrar por magic y symbol) - Si la posicion existe: llamar ApplyTrailingStop() - Si no existe: buscar en historial de deals el motivo de cierre (SL/TP/EA/MANUAL), logear, volver a IDLE INTERFAZ GRAFICA (modulo UI) Panel minimizable en esquina superior izquierda del chart: - Titulo: "NEWS STRADDLE EA v2.01" (dorado) - Estado actual (IDLE/ARMED/MONITORING/TRADE ACTIVE) - Hora del evento configurada - Countdown T-MM:SS (en estado ARMED) - Resumen de parametros: Dist | SL | TP (o OFF) | Trail (o OFF) - Boton CANCEL ALL (rojo) — cierra posicion y vuelve a IDLE - Boton minimizar "-" / "+" (esquina superior derecha del panel) HANDLERS PRINCIPALES OnInit(): - InitPipFactor(), SetExpertMagicNumber, SetDeviationInPoints(50) - Detectar filling mode (IOC/FOK/RETURN) y configurar CTrade - ParseEventTime() para convertir InpNewsTime a datetime - Estado inicial = STATE_ARMED - InitUI(), EventSetMillisecondTimer(500) OnTick(): - STATE_MONITORING: verificar si precio cruza niveles, ejecutar market order - STATE_TRADE_ACTIVE: buscar posicion por magic, aplicar trailing, detectar cierre OnTimer(): - STATE_ARMED: comparar TimeCurrent() con (eventTime - InpLeadTimeSeconds) Si llego el momento -> SetStraddleLevels() -> STATE_MONITORING - UpdateUI() siempre OnChartEvent(): - Detectar clicks en BTN_CANCEL (cerrar posicion, cleanup, IDLE) - Detectar clicks en BTN_MINIMIZE (ToggleMinimize) OnDeinit(): - CleanupUI() — eliminar todos los objetos graficos - EventKillTimer() ESTRUCTURA DE ARCHIVOS Arquitectura modular con includes: NewsStraddleEA.mq5 -- Motor principal (handlers) includes/Inputs.mqh -- Todos los input parameters includes/Globals.mqh -- Estado, variables globales, utilidades includes/UI.mqh -- Panel grafico, botones, labels includes/Orders.mqh -- Calculo de niveles, trailing stop CRITERIOS DE ACEPTACION - Compila sin errores ni warnings en MetaEditor 5. - Los niveles se calculan exactamente a T-LeadTimeSeconds del evento. - Se ejecutan market orders (NO ordenes pendientes) cuando el precio cruza niveles. - El trailing stop se activa solo al alcanzar breakeven, protegiendo ganancias. - El TP es opcional (InpEnableTP toggle). - El boton CANCEL ALL cierra posiciones y resetea a IDLE. - El panel es minimizable y muestra estado en tiempo real. - OnDeinit limpia todos los objetos graficos. - Todos los errores se manejan con logs claros (Print con tags [INIT], [MARKET], [TRAIL], etc.). - El EA no interfiere con otras posiciones (filtro por Magic Number).
